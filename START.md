# Инструкция по запуску Content Factory в чистой среде

Этот документ содержит пошаговое руководство по развертыванию и запуску всех сервисов проекта.

## 1. Системные требования
*   **Python:** Версия 3.9 или выше.
*   **FFmpeg:** Обязательно должен быть установлен в системе.

### Установка FFmpeg:
*   **macOS (Homebrew):**
    ```bash
    brew install ffmpeg
    ```
*   **Ubuntu/Linux:**
    ```bash
    sudo apt update
    sudo apt install ffmpeg
    ```

## 2. Подготовка окружения
1.  **Клонируйте проект** и перейдите в его корень.
2.  **Создайте виртуальное окружение:**
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    ```
3.  **Установите зависимости:**
    ```bash
    pip install --upgrade pip
    pip install fastapi uvicorn python-dotenv requests aiohttp aiogram watchdog
    ```
    *(Если есть файл requirements.txt, используйте `pip install -r requirements.txt`)*

## 3. Настройка конфигурации (`.env`)
Создайте файл `.env` в корне проекта и заполните его:
```env
# Пути к FFmpeg (если установлен в систему, просто "ffmpeg")
FFMPEG_BIN="ffmpeg"
FFPROBE_BIN="ffprobe"

# Настройки Telegram
TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН_БОТА"

# Базовые URL сервисов (по умолчанию)
CONTENT_API_BASE_URL="http://127.0.0.1:8002"
PUBLISH_API_BASE_URL="http://127.0.0.1:8010"
MEDIA_HTTP_BASE_URL="http://127.0.0.1:8080"
```

## 4. Запуск сервисов
Для стабильной работы рекомендуется запускать сервисы через watchdog-скрипты. Запустите каждую команду в отдельном окне терминала:

1.  **Медиа-сервер (хранилище):**
    ```bash
    ./scripts/run_media_http_watchdog.sh
    ```
2.  **API генерации контента (создание видео):**
    ```bash
    ./scripts/run_content_api_watchdog.sh
    ```
3.  **API публикации (интеграция с соцсетями):**
    ```bash
    ./scripts/run_publish_api_watchdog.sh
    ```
4.  **Telegram Бот (интерфейс управления):**
    ```bash
    ./scripts/run_bot_watchdog.sh
    ```

## 5. Важные рекомендации (решение проблем)
*   **Музыка:** Используйте файлы в формате **.mp3**. Система теперь поддерживает и .mp4, но .mp3 работает быстрее и надежнее.
*   **Шрифты:** Если вы запускаете проект на Linux, убедитесь, что путь к шрифту в `api/content_api.py` (строка ~1080) корректен. Обычно в Linux это `/usr/share/fonts/truetype/...`.
*   **Ошибки памяти:** Если FFmpeg выдает ошибку "No space left on device", проверьте наличие свободного места в папке `storage/tmp`.

## 6. История одной ошибки: "No space left on device"
Если при генерации видео вы видите ошибку `Error while filtering: No space left on device` при наличии свободного места на диске, знайте:
*   **Причина:** Баг FFmpeg при совместном использовании `-stream_loop -1` и `-shortest` на тяжелых входных файлах (например, если музыка — это видео `.mp4`). FFmpeg начинает бесконечно буферизировать данные, забивая временную память.
*   **Решение:** 
    1. Не использовать `-shortest`, вместо этого вычислять длительность через `ffprobe` и передавать её через `-t`.
    2. Всегда конвертировать фоновую музыку в `.mp3`.
    3. Использовать флаги `-vn -sn -dn` для музыкального входа, чтобы FFmpeg даже не пытался читать из него что-то, кроме звука.
